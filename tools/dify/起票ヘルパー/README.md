# 起票ヘルパー（Dify Workflow）

## 課題
バグ発見 → 起票までに時間がかかっていた。
- エンジニアが読みやすい文章を意識するあまり起票に時間がかかっていた
- 既存テンプレートは確認端末・環境を状況に応じて手動削除する必要があった

## 解決
Difyワークフローで入力を構造化し、ドラフトを自動生成。
ワンオペQAとして起票1件あたり約5分の削減を実現（実務投入済み）

## 入力
- 操作ログ（確認状況）
- 確認した結果 / 期待する結果
- 再現率
- 確認端末の選択
- 確認環境の選択

## 出力
- バグ報告文ドラフト（QCテンプレ / 要約 / 報告文 の3形式）
- 再現手順ドラフト
- 確認端末リスト（**デフォルト動作：未選択時は全端末を自動出力**）
- 確認環境リスト

## 使用技術
- Dify（ワークフロー）
- GPT-4o-mini
- Python（データ整形コード）
