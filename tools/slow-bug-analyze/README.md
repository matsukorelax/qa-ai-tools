# slow-bug-analyze

## 概要
自作バグチケットDBを分析し、「テスターが何を変えれば
バグ修正までのリードタイムを短縮できるか」を自動出力するパイプライン。

## 背景
スプレッドシートで約150件のバグチケットを蓄積する中で、
「起票の質がリードタイムに影響しているのでは」という仮説を持った。
テスター自身の行動改善につながる分析を自動化するために構築。

## システム構成
n8n（Webhook受信・バッチ処理）→ Dify（AI分析）→ Markdownレポート出力

## 処理フロー
1. バグチケットデータをWebhook経由でn8nに送信
2. チケットをループ処理しながらDifyへ1件ずつ投げる
3. Difyがバグ内容・重要度・リードタイム等を分析
4. 「テスターの改善アクション」をMarkdownテーブルで出力
5. 全件分を1つのファイルに集約

## 分析観点
- LeadTimeDaysが長いバグに共通する起票内容の傾向
- Category / Component / Severity別の改善ポイント
- DescriptionLenと調査遅延の相関

## 出力
優先度付きの改善アクション一覧（Markdownテーブル形式）

## 使用技術
- n8n（ワークフロー自動化・バッチ処理）
- Dify（AIワークフロー）
- Gemini 3 Pro Preview
- JavaScript（データ整形）
- 構造化出力（JSONスキーマ）

## データ
自作バグチケットDB（スプレッドシート・約150件）
